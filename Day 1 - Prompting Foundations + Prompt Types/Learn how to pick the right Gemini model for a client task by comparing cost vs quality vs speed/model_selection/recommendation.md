# Gemini Model Benchmark Report

## 1. Objective

The goal of this experiment is to compare three Gemini model tiers:

- gemini-3.1-pro-preview (highest accuracy, slowest, most expensive)
- gemini-2.5-flash (balanced performance)
- gemini-2.5-flash-lite (fastest, cheapest, lower quality)

We evaluate them across:
- Latency (speed)
- Token usage (efficiency)
- Cost (economic efficiency)
- Quality score (LLM-as-a-judge evaluation)

---

## 2. Task Description

We used a single standardized task:

> Summarize a 1-page document into exactly 3 bullet points.

This task is:
- Medium complexity
- Requires understanding + compression
- Sensitive to instruction following

We tested 20 diverse inputs including:
science, history, AI, biology, technology, and general knowledge.

---

## 3. Metrics Definition

### 3.1 Latency (ms)

Latency measures response time of the model.

Formula used:

- start_time = before API call
- end_time = after response received
- latency_ms = (end_time - start_time) × 1000

This measures real-world speed experienced by users.

---

### 3.2 Token Usage

Each API response returns token metadata:

- input_tokens → tokens used in prompt
- output_tokens → tokens generated by model

Extracted using:

response.usage_metadata.prompt_token_count  
response.usage_metadata.candidates_token_count

Tokens directly impact cost and performance.

---

### 3.3 Cost Calculation

Cost is computed using:

cost = (input_tokens × input_rate) + (output_tokens × output_rate)

Each model tier has different pricing:
- Pro → highest cost per token
- Flash → medium cost
- Flash-Lite → lowest cost

This allows fair economic comparison.

---

### 3.4 Quality Score (LLM-as-a-Judge)

We use Gemini 3.1 Pro as an evaluator model.

It scores outputs from 1 to 5 based on:

- accuracy
- clarity
- completeness
- conciseness

The judge prompt forces a single numeric output to ensure consistency.

This replaces human evaluation and enables scalable benchmarking.

---

## 4. Observations

### 4.1 Gemini 3.1 Pro Preview

- Highest quality summaries
- Strong reasoning and compression ability
- Best instruction following
- Higher latency compared to others
- Highest cost per request

Best suited for:
- critical tasks
- reasoning-heavy workloads
- high accuracy requirements

---

### 4.2 Gemini 2.5 Flash

- Strong balance of quality and speed
- Slightly less detailed than Pro
- Much faster than Pro
- Significantly cheaper

Best suited for:
- production workloads
- general summarization
- real-time applications

---

### 4.3 Gemini 2.5 Flash-Lite

- Fastest response time
- Lowest cost
- Occasionally misses fine details
- Slightly weaker instruction adherence in complex cases

Best suited for:
- high-volume tasks
- low-stakes applications
- large-scale automation pipelines

---

## 5. Overall Results Pattern

| Model | Latency | Cost | Quality |
|------|--------|------|--------|
| Gemini 3.1 Pro Preview | High | High | Best |
| Gemini 2.5 Flash | Medium | Medium | Good |
| Gemini 2.5 Flash-Lite | Lowest | Lowest | Moderate |

---

## 6. Final Recommendation

### Recommended Model: Gemini 2.5 Flash

### Reason:

After analyzing all 4 dimensions:

- Pro gives best quality but is expensive and slower
- Flash-Lite is cheap and fast but loses accuracy
- Flash provides optimal balance across all metrics

Therefore, for this summarization task:

👉 Gemini 2.5 Flash is the best production choice.

---

## 7. Decision Rules for Production Use

Use this mapping for real-world applications:

- High accuracy required (legal, finance, medical) → Gemini 3.1 Pro Preview
- General-purpose production workloads → Gemini 2.5 Flash
- High-volume + low-cost + low-stakes tasks → Gemini 2.5 Flash-Lite

---

## 8. Conclusion

This benchmark demonstrates the trade-off between cost, latency, and quality across Gemini model tiers.

Gemini 2.5 Flash emerges as the optimal model for document summarization due to its balanced performance across all evaluation metrics.
